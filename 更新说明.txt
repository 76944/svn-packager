============================================
  SVN增量打包工具 v1.0.0
  适用于 Tomcat 8.5
============================================

【系统要求】
- Windows 7/8/10/11（64位推荐）
- Java运行时环境 JDK/JRE 1.8 或更高版本
- 内存：至少512MB
- 硬盘：至少100MB

【安装方法】
1. 运行 SVNPackager_Setup.exe
2. 按照安装向导提示操作
3. 安装完成后，桌面会生成快捷方式

【首次使用】
1. 启动程序
2. 点击"新增"按钮配置SVN项目
3. 填写项目名称、SVN地址、本地路径
4. 填写SVN用户名和密码（如有）
5. 点击"测试连接"验证SVN连接
6. 保存项目配置

【使用步骤】
1. 在左侧项目列表中选择要操作的项目
2. 设置日期范围，点击"获取日志"获取SVN提交记录
3. 勾选需要打包的提交记录
4. 点击"预览变更文件"查看文件变更详情
5. 点击"打包"进入打包页面
6. 选择打包方式：
   - 编译项目：仅执行Maven编译
   - 全量打包：编译并打包所有文件为ZIP
   - 增量打包：仅打包变更的文件

【打包输出】
- ZIP文件默认输出到项目本地目录下的output文件夹
- ZIP文件命名格式：项目名_时间戳.zip
- 增量打包文件命名格式：项目名_incremental_时间戳.zip

【Tomcat 8.5部署】
- 将打包的ZIP文件解压到Tomcat的webapps目录
- 重启Tomcat服务即可

【配置文件位置】
- Windows: %APPDATA%\SVNPackager\config\config.json
- 其他系统: ~/.svn-packager/config.json

【常见问题】
Q: 提示"找不到JRE"怎么办？
A: 请下载并安装JDK 8，或使用捆绑JRE版本

Q: SVN连接失败？
A: 检查网络连接，确认SVN地址和凭证正确
   确认SVN服务器允许HTTP/HTTPS访问

Q: Maven编译失败？
A: 确认本地已安装Maven并配置环境变量
   确认项目根目录包含pom.xml文件

Q: 打包的文件在Tomcat中无法访问？
A: 确认打包的目录结构符合Tomcat webapps规范
   检查WEB-INF目录是否完整

  （仅当结束日期早于开始日期时才对齐，否则保留已选结束日期）

  currentProject 为 null 导致 setName 异常
  （保存前增加空值守卫，自动按"新增项目"处理）
- 修复日历星期表头错乱：表头"日 一 二 三 四 五 六"换行错位
  （将星期表头拆为独立的单行面板，不再与日期格子混排）
- 修复日历布局异常：星期表头行被过度拉伸、日期网格被挤压变形
  （日历主体改为 BorderLayout：表头固定高度置于 NORTH，
   日期网格填充 CENTER，并统一表头与日期标签的内边距）
- 修复日志列表选中提交记录后点击"打包"仍提示"请先选择要打包的提交记录"的问题
  （根因：表格第 0 列存的是 Long 版本号，原 getSelectedRecords() 把 Long 当 CommitRecord 处理并全部跳过；
   修复：按版本号到 commitRecords 中反查对应的 CommitRecord 并返回）
- 新增一键启动脚本 run.bat：自动使用系统已有 Java 运行 JAR，
  避免被迫安装特定版本的 Java 8
